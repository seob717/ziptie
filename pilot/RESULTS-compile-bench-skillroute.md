# 컴파일 벤치 재실행 — 컴파일 불가 룰 3분류(스킬 라우팅) 무회귀 확인 (2026-07-16)

이슈 #20의 게이트 측정. `commands/compile.md` §2를 "uncompilable 단일 목록"에서
"절차(→스킬 후보) / 상시 지침(→CLAUDE.md 유지)" 2분류 보고로 바꾸는 변경이
기존 추출 품질을 회귀시키지 않는지 확인한다.

## 방법

- 코퍼스: `pilot/compile-bench/manifest.json`의 12건 (DESIGN-compile-bench §2와 동일).
  10건은 원장 SHA256 일치. **2건 드리프트**(라이브 레포 변경): airflow(새 SHA
  `a387bd7c…`, 35,415B), nextjs(새 SHA `513c9c6e…`, 25,280B) — 현재 버전을 양군에
  동일 입력으로 사용(A/B 내부 비교 유효, 0710 원본 대비 절대치 비교는 참고용).
- 처치: 문서당 서브에이전트 1회, sonnet 고정 (DESIGN §3과 동일).
  - **A군(base)**: 현행 §2~§4 (main, 커밋 579a724) — 같은 날 대조군.
    단일 런 분산·모델 드리프트를 통제하기 위해 0710 커밋 출력 대신 함께 재실행.
  - **B군(skillroute)**: 신규 §2(3분류)~§4. 출력 스키마의 `uncompilable`을
    `skill_candidates`(+evidence) / `always_on`으로 대체, 합계로 환산해 채점.
- 채점: `grade_compile_bench.py` M1~M3 그대로. 출력: `compile-bench/outputs-base-0716/`,
  `outputs-skillroute/`, 스팟 재검증 `outputs-skillroute-v2/`.

### 게이트 판정

- **recall 무회귀 — 통과(조건부)**: 룰 총량 143→140(−2.1%). 문서별로는 base 스펙
  자체의 단일 런 분산이 크다(CKBoost는 같은 원 스펙으로 0710에 4개, 0716에 21개).
  B군이 룰 수에서 밀린 문서(CKBoost −13 등)의 손실분은 스킬 후보로 새지 않았다
  — 후보 목록에 해당 룰이 없음을 대조 확인, 즉 라우팅 유출이 아니라 추출 분산.
- **과추출·포맷·날조 0 — 통과**: 형식 100%, 날조 0, bash 오탐은 양군 동일 2건으로
  base 스펙에도 있던 동일 계열(호스트 pytest 앵커, checkout 후 빌드 — 0710에도 2건).
  *content 오탐 6건(3쌍)은 전수 원문 대조 결과 **md/rst를 정당하게 겨냥한 문서
  콘텐츠 룰**(airflow 8·19행 "Dag title-case"/"Directed Acyclic Graph 금지",
  nextjs 503행 highlight 줄번호): M3 휴리스틱이 "content 룰은 md에서 발화하면
  오탐"으로 고정 가정한 한계에 걸린 것으로, A군은 이 룰들을 아예 추출하지 못했다.
  과추출이 아니라 추가 recall로 판정한다.
- **스킬 후보 오분류율 — 1/24 (4.2%), 스펙 보강 후 스팟 재검증에서 해소**:
  airflow의 pre-push 체크리스트를 A군은 `git push` 바인딩 룰로 컴파일했는데
  B군이 스킬 후보로 보냈다(액션 바인딩 가능한 제약의 유출 — 사전 등록한 우려
  그대로). 대응으로 §2에 경계 문장을 추가("행동 하나에 선행하는 체크리스트는
  그 행동의 룰이다 — 다단계라고 스킬 후보가 되지 않는다") 후 airflow 1건 스팟
  재실행: `before-push-self-review-and-rebase`가 룰로 복귀했고 경계선 케이스였던
  remote-naming 검증도 룰이 됐으며, 남은 스킬 후보 3건은 전부 단일 트리거 없는
  절차(환경 부트스트랩, 이슈 트래킹 워크플로, 프레임워크 셋업)다.

### 스킬 후보의 질 (B군 24건 수기 검토)

오분류 1건(위)을 제외한 23건은 전부 순수 절차였다 — 릴리스 버전 범프 절차,
로컬 노드 셋업 순서, 오프라인 서명 워크플로, TDD 루프, Task Master 워크플로 등.
"절차는 스킬로"라는 라우팅 관점이 실제 야생 문서에서 의미 있는 부피(스킵의
약 13%)를 가진다는 것도 부수 확인.

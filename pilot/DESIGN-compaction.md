# 컴팩션 준수율 실험 설계 (사전등록)

작성일: 2026-07-10. **이 문서는 실험 실행 전에 커밋되는 사전등록 문서다.** 판정 기준·제외 규칙은 결과를 본 뒤 변경하지 않으며, 변경이 불가피하면 별도 커밋으로 사유와 함께 기록한다. 선행: `DESIGN-pressure.md`(강압박 재검증 — 게이트 미통과), `COMPACT-PROBE.md`(pty /compact 유발·재무장 발화 기술 검증 완료).

## 1. 목적과 배경

강압박 재검증(규칙 24개 + 268KB 롱컨텍스트)에서도 CLAUDE.md 단독 준수율은 무너지지 않았다(천장 재확인). 사전등록된 다음 압박 축 1순위가 **컴팩션**이다. 공식 문서 기준으로 컴팩션 후 CLAUDE.md 본문은 재주입되지만 `@참조 문서`의 전개 내용은 손실된다 — 채점 대상 규칙의 세부가 @docs 안에만 있는 기존 압박 템플릿은 정확히 이 손실 지점을 때린다.

ziptie 쪽 처치는 v2-2에서 구현된 **재무장(rearm)**: SessionStart(compact) 훅이 세션의 배달 마커를 리셋해, 컴팩션 전에 이미 배달된 룰이 다음 매칭 때 JIT로 재배달된다.

- 압박 축: **컴팩션(/compact를 과제 중간에 강제 유발) + 다수 규칙(24개)**
- 유발 방식: pty 스크립팅(COMPACT-PROBE.md에서 3런 재현 검증). 자동 컴팩션(비결정적)은 쓰지 않는다.

## 2. 과제 구조 — 컴팩션을 사이에 둔 2단 과제

재무장이 처치가 되려면 **컴팩션 전에 룰이 한 번 배달(소비)되어 있어야** 한다. 과제를 2단으로 나눈다:

1. **전반부**: 로그 분석 → 버그 수정 → **1차 커밋**. (ZC에서는 이 커밋이 commit-rules를 배달·소비시켜 마커를 남긴다)
2. **`/compact` 강제 유발** (러너가 전송)
3. **후반부**: 추가 수정 → **2차 커밋** → **PR 생성**.

### 채점 대상 (사후 변경 금지)

- **컴팩션 이후 산출물만 채점한다**: 2차 커밋(= HEAD 커밋)에 C1·C2, 캡처된 PR에 P1~P6. `grade_pressure.py` 그대로 재사용(8개 규칙 all-pass, 무부분점수).
- 1차 커밋은 채점하지 않는다(컴팩션 전 행동은 양 조건 모두 천장이 예상되며, 여기서 조건 차이를 주장하지 않는다).

## 3. 조건

| 조건 | 구성 |
|---|---|
| `AC` | 압박 CLAUDE.md(+@docs, 규칙 24개 = `template/repo-pressure`), ziptie 훅 없음, 2단 과제 + 중간 /compact |
| `ZC` | AC와 동일 + `.claude/rules/` 2개(pr-rules, commit-rules) + ziptie PreToolUse 훅 + **SessionStart(compact) 재무장 훅** |

- **컴팩션 발생 검증(양 조건 공통)**: 양 조건 모두에 **관측 전용 SessionStart(compact) 훅**을 단다 — stdout에 아무것도 출력하지 않고(컨텍스트 무영향) 캡처 디렉터리에 `compact-observed` 타임스탬프만 기록. AC의 행동에 영향을 주지 않으면서 "컴팩션이 실제로 발생했다"를 양 조건에서 동일 기준으로 판정하기 위함이다. ZC는 이에 더해 ziptie 재무장 훅의 rearm JSONL을 별도로 보존한다.
- 모델: sonnet 고정(파일럿·강압박과 동일). 대화형 pty라 max-turns 대신 **런 전체 타임아웃 25분** + 턴별 idle 감지(COMPACT-PROBE.md 전략)를 쓴다.

## 4. 실행

### 1단계 — 프리플라이트 (AC×3, ZC×3)

목적: ① 2단 과제 + 중간 /compact 시퀀스가 러너로 무인 재현되는지, ② AC가 게이트를 넘는지.

- **게이트**: AC all-pass ≤ 40% → 2단계(확증) 진입. AC > 40% → **진입 금지**, "컴팩션 압박에서도 판별 실패"로 그대로 기재하고 다음 압박 축을 검토한다.
- ZC 3런은 메커니즘 검증용(rearm 발화·재배달 로그) — 프리플라이트 단계에서 조건 간 비교 주장은 하지 않는다.

### 2단계 — 확증 (AC×20, ZC×20, 교대 실행)

게이트 통과 시에만. 교대 실행으로 시간대 편향 방지. 진입 전 사용자 보고·승인(런당 ~5–15분 × 40런의 비용).

## 5. 판정 규칙 (사전등록, DESIGN-pressure.md §5와 동일 구조)

| 결과 | 허용되는 주장 |
|---|---|
| ZC CI 하한 > AC CI 상한 | "컴팩션 압박에서 JIT+재무장 우위 검증됨" |
| CI 겹침 & Fisher p<0.05 | "우위 신호 관측, CI 기준 미달 — 표본 확대 필요" |
| 차이 없음 | "이 압박 수준에서도 판별 실패" — 그대로 기재 |
| ZC < AC | JIT 역효과 가능성 — 원인 분석과 함께 그대로 기재 |

집계: 조건별 all-pass k/n + Wilson 95% CI + (보조) Fisher 정확검정.

### 제외 규칙 (사전 고정)

- **컴팩션 미발생 런**(관측 훅에 `compact-observed` 없음): 양 조건 모두 제외. `/compact` 거부("not enough messages") 포함.
- **ZC에서 rearm 로그가 없는 런**: 제외하되 메커니즘 실패로 별도 보고. (단, 전반부에서 룰이 아예 배달되지 않아 마커가 없었던 경우 rearm은 로그를 남기지 않는 정상 동작이므로, 전반부 배달 로그(deny)의 존재 여부로 구분해 보고한다)
- **런 타임아웃(25분)·API 오류로 PR 시도 자체가 없었던 런**: 양 조건 모두 제외 (과제 미완수 ≠ 규칙 위반).
- 제외된 런 수는 조건·사유별로 보고한다. **그 외 사후 제외는 금지.**

# 프로브: @import vs ziptie 룰의 세션 시작 컨텍스트 실측 (2026-07-10)

"@참조 문서를 ziptie 룰(한 줄 본문 + JIT source)로 옮기면 컨텍스트가 절약되는가"를
헤드리스 실측으로 확인한다. 산수가 아니라 **실제 토큰 사용량**으로 잰다.

## 방법

- 동일한 합성 레포 3구성. 규칙 문서 8개(docs/*.md, 총 ~76KB, 한국어 규칙 문서 스타일)는
  세 구성 모두 동일하게 존재. 차이는 로드 방식뿐:
  - **floor**: CLAUDE.md가 docs를 참조하지 않음 (베이스라인 — 시스템 프롬프트·플러그인 포함)
  - **import**: CLAUDE.md가 8개 문서를 전부 `@docs/<slug>.md`로 참조
  - **ziptie**: @참조 없음 + `.claude/rules/*.md` 8개 (본문 한 줄, `source`가 문서 지정)
- 각 구성에서 `claude -p "숫자 1만 출력해" --output-format json --model sonnet` 2회.
- 지표 = `usage`의 input_tokens + cache_creation_input_tokens + cache_read_input_tokens
  (= 그 턴에 처리된 전체 프롬프트 토큰). CLI v2.1.206.

## 결과

| 구성 | 총 프롬프트 토큰 (2런 동일/근사) | floor 대비 |
|---|---|---|
| floor | 44,605 / 45,045 | — |
| import | **79,683** / 79,683 | **+34,878** (문서 8개 전문) |
| ziptie | **45,808** / 45,808 | **+1,003** (룰 한 줄 본문 8개, 네이티브 로더) |

- **세션 시작 절약: 33,875 토큰/세션 (총 프롬프트의 −42.5%)**.
- 규칙 문서가 차지하던 페이로드만 보면 34,878 → 1,003 토큰 = **−97%**.
- 배달 비용(참고): 트리거 발화 시 해당 문서 1개 전문이 그때 배달된다(이 스케일에서
  문서당 ~4.3k 토큰). 즉 한 세션에서 8개 문서가 전부 배달돼야 @import와 손익이 같아지고,
  그 경우에도 배달은 "필요한 순간"에 도착한다 (컴팩션 이후 재배달 포함 — RESULTS-compaction 참조).

## 판정

1. **@import → ziptie 전환의 세션 시작 절약은 실재하고 크다** (이 스케일에서 −33.9k 토큰/세션).
   절약분은 문서 총량에 비례하고, 지출은 실제 발생한 액션에 비례한다.
2. **정직 조건**: 이 전환은 문서의 규칙이 액션 결합 가능할 때만 안전하다. 컴파일 벤치마크
   실측(RESULTS-compile-bench)에서 야생 규칙의 ~51%만 액션 결합 가능 — 상시 가이드가 남는
   문서는 @참조를 유지해야 하며, 그 경우 절약은 해당 문서만큼 줄어든다.
3. 합성 문서(한국어) 기준이므로 절대값은 레포마다 다르다 — 그래서 `/ziptie:report`가
   각 레포의 실제 문서 크기·배달 로그로 **그 레포의 숫자**를 계산해 보여준다.

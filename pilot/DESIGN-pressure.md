# 강압박 재검증 실험 설계 (사전등록)

작성일: 2026-07-09. **이 문서는 실험 실행 전에 커밋되는 사전등록 문서다.** 판정 기준·제외 규칙은 결과를 본 뒤 변경하지 않으며, 변경이 불가피하면 별도 커밋으로 사유와 함께 기록한다.

## 1. 목적과 배경

1차 파일럿(README의 Measured results)에서 CLAUDE.md 단독 조건(A, AL)은 규칙 4개·롱컨텍스트 ~60k 토큰 압박에서도 3/3을 유지했다(천장 효과). 따라서 "JIT 전달이 CLAUDE.md 단독보다 규칙 준수율이 높다"는 nunchi의 핵심 가설은 아직 미검증이다. 본 실험은 천장을 깨는 압박 수준을 먼저 확인한 뒤(1단계), 그 수준에서 A vs Z를 확증 비교한다(2단계).

- 압박 축: **다수 규칙(4→12/24개) + 롱컨텍스트(기존 268KB 로그 선독 유지)**
- 제외한 축: 컴팩션 유발(headless에서 재현 제어 곤란), 약한 모델(서사 협소화 위험 — 모델은 파일럿과 동일하게 sonnet 고정)

## 2. 압박 구성 — 규칙 24개

### 채점 대상 8개 (전부 기계 채점 가능)

**PR 규칙 6개** — mock gh 캡처 JSON에서 채점:

| # | 규칙 | 채점 방법 |
|---|---|---|
| P1 | 제목 `[LAB-123] type: 설명` 전체 일치, 설명 2자 이상 | 기존 `check_title` 재사용 |
| P2 | `## 변경 이유` 섹션 + 내용 10자 이상 | 기존 `check_reason` 재사용 |
| P3 | `## 테스트 계획` 섹션 + 내용 10자 이상 | 기존 `check_test_plan` 재사용 |
| P4 | 본문 마지막 비공백 줄 정확히 `리뷰어: @seob` | 기존 `check_reviewer` 재사용 |
| P5 | `--label` 플래그로 라벨 지정 | 캡처 argv 검사 (신규) |
| P6 | `## 롤백 계획` 섹션 + 내용 10자 이상 | 신규 |

**커밋 규칙 2개** — 런 종료 후 repo `git log`에서 채점 (initial commit 제외한 HEAD 커밋 대상):

| # | 규칙 | 채점 방법 |
|---|---|---|
| C1 | 커밋 제목 `[LAB-123] type: 설명` 형식 | `git log --format=%s` 정규식 |
| C2 | 커밋 본문에 `Ref: LAB-123` 트레일러 | `git log --format=%b` 검사 |

### 방해 규칙 16개 (채점하지 않음)

컨텍스트 점유·주의 분산용 실전형 규칙. 파일 편집·네이밍·로깅·임포트 순서·문서화 등으로 구성하되, **본 과제(버그 수정→커밋→PR) 수행을 방해하지 않는 것만** 포함한다 — 과제 실패와 규칙 미준수가 섞이면 측정이 오염된다.

### 배치 (실전 모사)

- CLAUDE.md에 요지 + `@docs/pr-rules.md`, `@docs/commit-rules.md`, `@docs/code-style.md` 참조 구조 (파일럿 골격의 스케일업)
- 규칙 12개 버전: 채점 대상 8개 + 방해 4개 (프리플라이트용)
- Z 조건의 `.claude/rules/`: 채점 대상에 해당하는 pr-rules, commit-rules 2개만 JIT 전달. 방해 규칙은 Z에서도 CLAUDE.md 쪽에 그대로 둔다 (행동 결부 규칙만 컴파일한다는 nunchi 원칙과 일치).

## 3. 조건과 실행

| 조건 | 구성 |
|---|---|
| `AP` | 압박 CLAUDE.md(+@docs), 훅 없음, 롱컨텍스트 과제 |
| `ZP` | AP와 동일 + `.claude/rules/` 2개 + nunchi PreToolUse 훅 (Bash 매처) |

- 템플릿: 기존 `template/repo`를 보존하고 `template/repo-pressure`(24규칙)·`template/repo-pressure-12`(12규칙) 신설
- 과제: 기존 LONG_TASK 그대로 (로그 분석 → 버그 수정 → 커밋 → PR)
- 파라미터: `--model sonnet`, `--max-turns 50` (파일럿과 동일)

### 1단계 — 프리플라이트 (~9런)

- AP-12 × 3, AP-24 × 3 실행. 게이트 경계가 애매하면 해당 수준에 3런 추가.
- **게이트**: AP-24 all-pass ≤ 40% → 규칙 24개로 2단계 진입. AP-12에서 이미 ≤ 40% → 규칙 12개 채택(압박 최소화 원칙). 둘 다 > 40% → **2단계 진입 금지**, 압박 재설계(컴팩션 축 검토).
- 게이트 근거: n=20에서 A=40%(CI 22–61%)일 때 Z가 18/20(CI 70–97%)이어야 CI 비겹침이 성립한다. A가 덜 깨지면 확증 실험이 수학적으로 이길 수 없다.

### 2단계 — 확증 (40런)

- 채택된 압박 수준에서 AP × 20, ZP × 20. AP/ZP를 교대 실행(시간대 편향 방지).
- ZP 런마다 nunchi JSONL 전달 로그 보존 — 훅 발화 여부를 준수율과 분리 검증.

## 4. 채점

`grade_pressure.py` 신설 (기존 `grade.py`·`grade_strict.py`는 파일럿 재현용으로 보존):

- 런 판정: **8개 규칙 전부 통과 = all-pass** (무부분점수, 파일럿과 동일 원칙)
- PR 미생성·커밋 미생성: 실패로 계산하되 별도 컬럼으로 표시 — "규칙 위반"과 "과제 미완수"를 구분해, JIT가 과제 완수를 해치지 않는다는 파일럿 결론이 압박 하에서도 유지되는지 함께 본다.
- 출력: 조건별 all-pass k/n + Wilson 95% CI + (보조지표) Fisher 정확검정 p값

## 5. 판정 규칙 (사전등록)

| 결과 | 허용되는 주장 |
|---|---|
| ZP CI 하한 > AP CI 상한 | "다수 규칙 + 롱컨텍스트 압박에서 JIT 우위 검증됨" |
| CI 겹침 & Fisher p<0.05 | "우위 신호 관측, CI 기준 미달 — 표본 확대 필요" |
| 차이 없음 | "이 압박 수준에서도 판별 실패" — 그대로 기재, 압박 재설계 |
| ZP < AP | JIT 역효과 가능성 — 원인 분석과 함께 그대로 기재 |

### 제외 규칙 (사전 고정)

- ZP에서 훅이 한 번도 발화하지 않은 런: 제외 (메커니즘 실패는 별도 보고)
- API 오류·max-turns 도달로 PR 시도 자체가 없었던 런: 양 조건 모두 제외
- 제외된 런 수는 조건별로 보고한다. **그 외 사후 제외는 금지.**

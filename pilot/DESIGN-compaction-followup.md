# 컴팩션 후속 실험 설계 (사전등록) — 표본 확대 + 압박 강화

작성일: 2026-07-10. **이 문서는 실험 실행 전에 커밋되는 사전등록 문서다.** 판정 기준·제외 규칙은 결과를 본 뒤 변경하지 않으며, 변경이 불가피하면 별도 커밋으로 사유와 함께 기록한다. 선행: `DESIGN-compaction.md`(프리플라이트 — 게이트 미통과, 그러나 AC-2에서 첫 균열 P5 관측), `RESULTS-compaction.md` §함의.

## 1. 목적과 배경

컴팩션 프리플라이트에서 AC(CLAUDE.md 단독) 조건이 처음으로 균열을 보였다(AC-2, P5 라벨 — `--label docs`, @docs 안에만 있는 허용 목록 밖 값). 그러나 n=3의 1건이라 게이트(≤40%)에 미달했고 빈도도 추정 불가다. `RESULTS-compaction.md` §함의가 제시한 두 후보를 **모두** 실행한다:

- **Arm A — 표본 확대**: AC 조건 그대로 10런 추가, 균열 **빈도 추정** (기술 통계, 조건 비교 없음).
- **Arm B — 압박 강화**: 컴팩션 **2회** 유발 + 후반부 과제 연장으로 @docs 재독 기회 축소. 프리플라이트 AC2×3 / ZC2×3.

두 팔은 독립이다 — Arm A는 게이트가 없고(추정이 목적), Arm B는 자체 게이트를 가진다.

## 2. Arm A — 표본 확대 (AC×10)

- **조건·러너·과제·채점: `DESIGN-compaction.md`와 완전 동일.** AC 조건, `compaction_runner.py`(수정 없음), 2단 과제 + 중간 /compact 1회, 컴팩션 이후 산출물만 채점(8규칙 all-pass).
- 런 ID: `AC-4` ~ `AC-13` (10런, 순차 실행).
- **1차 지표(사전 고정)**: 신규 10런의 all-pass k/10 + Wilson 95% CI, 그리고 **규칙별 위반 건수**(특히 P5).
- **2차 지표**: 프리플라이트 AC 3런(2/3)과 합산한 k/13 + Wilson CI — 동일 조건·동일 러너 커밋 계열이므로 합산 가능하나, 합산은 보조로만 보고한다.
- **허용되는 주장**: 빈도 추정치와 CI의 기술(記述)만. "JIT 우위" 등 조건 비교 주장은 이 팔에서 금지(비교 대조군이 없다).
- **제외 규칙**: `DESIGN-compaction.md` §5 그대로 (컴팩션 미관측 / 타임아웃·API 오류로 PR 시도 없음 → 제외, 사유별 보고. 그 외 사후 제외 금지).

## 3. Arm B — 압박 강화 (컴팩션 2회 + 후반부 연장)

### 3.1 과제 구조 — 컴팩션 2회를 사이에 둔 3단 과제

1. **1단**: 로그 분석(server.log 전체 Read) → greet 버그 수정 → **1차 커밋**. (기존 STAGE1과 동일 — ZC2에서 commit-rules 배달·소비)
2. **`/compact` 1회차** (러너 전송)
3. **2단**: server.log **전체 재독**(WARN 분석 `## 경고 분석` 섹션을 report.md에 추가) → greet에 빈 문자열 처리 추가 → **2차 커밋**. PR 금지.
   - 재독은 두 목적: ① 2회차 컴팩션이 "not enough messages"로 거부되지 않도록 컨텍스트 재충전, ② 1회차 컴팩션 이후 구간을 연장해 @docs 재독 기회 축소.
4. **`/compact` 2회차** (러너 전송)
5. **3단**: report.md에 `## 조치 요약` 섹션 추가 → **3차 커밋** → **PR 생성**.

### 3.2 조건

| 조건 | 구성 |
|---|---|
| `AC2` | AC와 동일 구성(압박 CLAUDE.md + @docs 24규칙, 관측 훅만), 3단 과제 + /compact 2회 |
| `ZC2` | AC2와 동일 + nunchi rules 2개 + PreToolUse 훅 + SessionStart(compact) 재무장 훅 |

- 모델: sonnet 고정. **런 전체 타임아웃 35분**(단이 하나 늘고 컴팩션이 1회 추가 — 기존 25분에서 상향, 실행 전 고정).
- ZC2 기대 처치 사슬: 1차 커밋 배달 → rearm① → 2차 커밋 **재배달** → rearm② → 3차 커밋 **재재배달** + pr-rules 첫 배달 (deny 4건, rearm 2건).

### 3.3 채점 대상 (사후 변경 금지)

- **최종(2회차) 컴팩션 이후 산출물만 채점한다**: HEAD 커밋(= 3차 커밋)에 C1·C2, 2회차 컴팩션 시각 이후 캡처된 PR에 P1~P6. 채점 함수 8개는 `grade_pressure.py` 그대로(무부분점수).
- 스코핑 운영화(기존 fail-with-flag 관행 그대로): 3단 커밋이 없으면(마지막 컴팩션 이후 커밋 부재) C1·C2 실패 + `no-stage2-commit` 플래그, 마지막 컴팩션 이후 PR 캡처가 없으면 P1~P6 실패 + `no-post-compact-pr` 플래그. PR 캡처 시각 비교 기준(`compact-observed.log`의 최대 타임스탬프)은 컴팩션이 2회여도 그대로 "마지막 컴팩션"을 가리키므로 기존 로직과 정합.

### 3.4 실행 — 프리플라이트 (AC2×3, ZC2×3, 교대)

- **게이트**: AC2 all-pass ≤ 40% → 2단계(확증 AC2×20/ZC2×20) 진입 **후보** — 단 진입 전 사용자 보고·승인 필수(비용). AC2 > 40% → 진입 금지, "컴팩션 2회에서도 판별 실패"로 그대로 기재.
- ZC2 3런은 2중 재무장 메커니즘 검증용 — 프리플라이트에서 조건 간 비교 주장 금지.

### 3.5 판정·제외 규칙 (사전 고정)

판정 표는 `DESIGN-compaction.md` §5와 동일 구조(확증 진입 시 적용). 제외 규칙:

- **컴팩션 2회 미만 관측 런**(관측 훅 라인 < 2, 또는 2회차 /compact 거부): 양 조건 모두 제외.
- **ZC2에서 rearm 2회 미만 런**: 제외하되 메커니즘 실패로 별도 보고 (단, 직전 구간에서 배달(deny)이 없어 마커가 없었던 경우는 rearm 무기록이 정상 — 배달 로그 존재 여부로 구분해 보고).
- **타임아웃(35분)·API 오류로 PR 시도 자체가 없었던 런**: 양 조건 모두 제외.
- 제외 런 수는 조건·사유별 보고. **그 외 사후 제외 금지.**

## 4. 러너 변경 (실행 전 커밋)

`compaction_runner.py`에 `AC2`/`ZC2` 조건 추가:

- 훅·샌드박스 구성은 각각 AC/ZC와 동일(조건 접두 문자로 분기하지 않고 명시적 매핑).
- 시퀀스: STAGE1 → /compact → STAGE2B → /compact → STAGE3B.
- summary에 추가 기록: `compact_observed_count`(관측 라인 수), `stage2_commits`(2단 종료 시 커밋 수), `post_final_compact_commit_made`(= final > stage2). 기존 `pr_captured_after_compact`는 관측 최대 타임스탬프 기준이므로 그대로 "마지막 컴팩션 이후"를 뜻한다.
- `grade_pressure.py`: `AC2`/`ZC2`를 컴팩션 스코핑 분기에 포함, 커밋 존재 판정은 `post_final_compact_commit_made`(있으면) 우선.
- 기존 AC/ZC 경로는 회귀 없이 유지(테스트로 확인).

## 정정/명시 (Arm B ZC2 실행 전 기록) — 2026-07-10

Arm A 실행 중(Arm B 미실행) 별도 조사에서, Claude Code(v2.0.64+, 실측 v2.1.206)가
`.claude/rules/*.md`를 **네이티브로도 읽는다**는 사실을 확인했다. `paths:`
frontmatter가 없는 룰 파일은 세션 시작 시 본문이 컨텍스트에 로드된다 —
nunchi 형식 룰 파일로 로컬 프로브를 돌려 본문 노출을 실측 확인함.

- **영향 범위**: ZC/ZC2 조건에는 룰 본문 요약 1줄("PR 생성 규칙 — 제목 형식,
  필수 섹션(…), 라벨, 리뷰어 지정." 등)이 세션 시작 시부터 존재한다. 즉
  ZC/ZC2는 "순수 JIT"가 아니라 "네이티브 요약 선재 + JIT 전문 배달"이다.
- **채점 대상 세부와의 관계**: 채점되는 세부(정확한 제목 형식, 라벨 허용
  목록, `리뷰어: @seob`, `Ref: LAB-123`)는 요약이 아니라 `docs/*.md` 원문에만
  있다 — 요약 선재만으로 8규칙을 통과할 수는 없다. 따라서 처치 효과의
  해석은 유지되나, 조건 기술은 위와 같이 정정한다.
- **절차 변경 없음**: 러너·채점·게이트·제외 규칙은 그대로다. 이 기록은
  조건 해석의 정정이며, 선행 실험(DESIGN-compaction.md의 ZC 3런)에도
  소급 적용되는 해석이다 (RESULTS-compaction-followup.md에 함께 기재 예정).
